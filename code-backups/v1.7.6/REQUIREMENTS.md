# Requirements & Specifications — PaySick v1.7.6

**Version**: 1.7.6
**Date**: 2026-04-20

Carries forward all requirements from v1.7.5 with the following additions.

---

## New Requirements

### Investor Deck — Universal Problem Framing

| ID | Requirement | Priority |
|----|-------------|----------|
| DECK-01 | The problem slide must frame the healthcare payment gap as a universal customer problem before presenting South African market statistics | Must Have |
| DECK-02 | The problem slide must present three stakeholder-level failure points: patient, provider, and funder | Must Have |

### Investor Deck — Shield™ as Proprietary AI Risk IP

| ID | Requirement | Priority |
|----|-------------|----------|
| DECK-03 | The investor deck must contain a dedicated slide positioning Shield™ as PaySick's primary AI-based proprietary risk IP | Must Have |
| DECK-04 | The Shield slide must enumerate all five Shield v5.0 Tariff Billing Controls with investor-accessible descriptions | Must Have |
| DECK-05 | The Shield slide must describe the Four-Gate Assessment system (Provider, Patient, Lender, Outcome) | Must Have |
| DECK-06 | The Shield slide must describe the two-stage 80%/20% disbursement architecture and circuit breaker controls | Must Have |
| DECK-07 | The Shield Intelligence Engine slide must reference Shield™ by name as the underwriting engine and describe its AI-driven, pre-disbursement controls | Must Have |

### Investor Deck — Two-Part Funding Ask

| ID | Requirement | Priority |
|----|-------------|----------|
| DECK-08 | The funding ask slide must present two structurally distinct capital tranches, not a single blended raise | Must Have |
| DECK-09 | Part 1 must describe a Disbursement Facility (institutional capital, senior debt / revolving facility, non-dilutive, ~3× capital turns per year) | Must Have |
| DECK-10 | Part 2 must describe an Equity Seed Round (operational run-rate capital, 18-month runway) | Must Have |
| DECK-11 | The equity offered figure must reflect only the equity tranche (Part 2: R10M / R60M post-money = 17%), not the combined total | Must Have |
| DECK-12 | The PPTX download function must mirror all slide content changes in the corresponding generation code | Must Have |
